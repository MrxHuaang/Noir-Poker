## Deferred — needs a design call, not auto-fixed

These are real but changing them alters visuals/UX; left for an explicit decision.

1. **Semantic feedback colors** — `NormalConfigPanel.tsx:198-199` uses `amber` (warn) /
   `emerald` (success) for the autosave feedback toast. The brand rule forbids amber/emerald
   chrome, but there is no warning/success token in the accent system, and forcing both to
   violet would erase the warn-vs-success distinction. Recommend adding semantic
   `--color-warn-*` / `--color-success-*` tokens to `globals.css` + `brand.ts` rather than
   flattening to accent. See future feature F1.
2. **Rank identity colors** — `RankTowerModal.tsx:26,44` hardcode `text-amber-500` / `text-yellow-400`
   per rank. Arguably cosmetic per-rank identity (like the exempt card faces), but not listed
   as exempt in CLAUDE.md. Decide: add ranks to the exempt list, or re-skin to accent shades.
