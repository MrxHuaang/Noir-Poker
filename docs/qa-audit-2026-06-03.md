### Intentionally left as literals (not dumb duplication)

- **Bespoke tier ramps** — `page.tsx` `TIERS` and `Nav.tsx` `ACCENTS` define cohesive
  `one/two/three` gradient scales; only tier `one` happens to equal `ACCENT_GLOW_COLORS`, and
  tiers two/three are off-ramp tints that would not track a re-skin anyway. Importing the const
  into one of three siblings hurts readability more than it helps. Left whole.
- **Confetti / card-back gradients** — mix off-ramp stops (`#c4b5fd`, `#f4f4f5`) not present in
  the accent ramp; substituting would change visuals. Left as-is.
- **Percent formatting** — `EquityBadge.formatPct` (input 0..1, 1 decimal, 100%/0% special-cased)
  and the outs `%` (input 0..100, 0 decimals) are semantically different formats with different
  domains. Unifying risks subtle off-by-domain bugs for near-zero DRY gain. Left separate; if
  consolidated later, give each its own named helper rather than one overloaded function.

## Future features

- **F1 — Semantic state tokens. DONE.** `warn-*` / `success-*` scales shipped (see Color
  decisions above). `info-*` not added yet — add the same way if an informational state appears.
- **F2 — Per-room stats/history in Firestore. NOT auto-shipped (deliberate).** `usePlayers` /
  `useStats` / `useHistory` are host-device-local localStorage. Migrating to `rooms/{code}`
  subcollections is genuinely the highest break-risk item in the backlog: it touches the data
  model, requires **`firestore.rules` changes (security-sensitive)**, and cannot be verified
  without live writes against the real Firebase project. Per the "don't break anything"
  constraint this was left for a supervised session. Implementation sketch below.
- **F3 — Turn timebank. ALREADY COVERED.** Live `TurnTimer.tsx` already runs a two-deadline
  model (`useCountdown(deadline)` + `useCountdown(totalDeadline)`). The removed `useTurnTimer`
  was an abandoned parallel attempt, not the live path. No further work unless a separate
  per-player bank (distinct from `totalDeadline`) is actually desired.
- **F4 — Shared utils. DONE** (`lib/num.ts`). A `lib/format.ts` for percent helpers was
  intentionally skipped — see the percent-formatting note above.

### F2 implementation sketch (for a supervised session)

1. Subcollections under `rooms/{code}`: `stats/{playerId}`, `history/{handId}` (cap + paginate,
   mirror the local 50-item cap).
2. Helpers in `src/lib/rooms.ts` (NOT direct `getFirestore()` in hooks) — `writeHandHistory`,
   `subscribeRoomStats`. Funnel writes through one place, like the existing state-sync effect.
3. `firestore.rules`: only `hostUid` writes stats/history; any room member reads. Test in the
   emulator before touching prod rules.
4. Migration: dual-read (Firestore first, localStorage fallback) so existing host devices keep
   working during rollout. Remove the local path only after confirming parity.
